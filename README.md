# Cloud-IT-fogalmak
## Skálázás, terheléselosztás, hibatűrés
A legkedvesebb témakör az alapfogalmak közül a **skálázás (scaling)**  fontossága, hogy a rendszer kapacitása igény szerint csökkenthető/növelhető akár a megnövekedett forgalom, felhasználók száma miatt vagy legyen szó napszakos vagy szezonális terhelésről vagy az üzleti igények bővülésével több feladatot kell ellátnia a szervernek. 

*1. Vertikális skálázás* a szerver gép teljesítmény növelése (CPU,RAM9).

*2. Horizontális skálázás* több gép bevonása párhuzamosan, elosztva a terhelést.

Ami ebben a legjobban tetszik az előretervezés és tudatos felépítése, hogy egyszerre legyen magas rendelkezésreállás *(HA)* és olyan tartalék elemek bevonás, amely egy hiba esetén is a szerver biztonságos működését támogatja *(redudancia).
